Add a new module called "Smart Clipper" to the existing AutoFlow application.
This module adds video clipping, subtitle generation, and manual download control.

---

## NEW MODULE: SMART CLIPPER

### WHAT IT DOES
Given a video file (uploaded directly) or a video URL (YouTube, TikTok, Instagram),
the system:
1. Downloads or receives the video
2. Analyzes it with AI to detect the most important/viral clip
3. Cuts that clip automatically
4. Adds burned-in subtitles to the clip
5. Gives the user full control to preview and download the final clip

### STEP 3 — CUT THE VIDEO
```javascript
async function cutClip(inputPath, startTime, endTime, jobId) {
  const outputPath = `/tmp/clip_${jobId}.mp4`;
  // FFmpeg command:
  // ffmpeg -ss <startTime> -to <endTime> -i input.mp4
  //        -c:v libx264 -c:a aac -avoid_negative_ts make_zero
  //        -movflags +faststart output.mp4
  // Use fluent-ffmpeg with progress events → emit to job status
  return outputPath;
}
```

### STEP 4 — GENERATE & BURN SUBTITLES
```javascript
async function addSubtitles(clipPath, transcript, startTime, endTime, jobId) {
  // 1. Filter transcript segments that fall within [startTime, endTime]
  //    Re-offset timestamps: segment.start -= startTime

  // 2. Generate .SRT file from filtered segments
  //    Format: index \n HH:MM:SS,ms --> HH:MM:SS,ms \n text \n\n

  // 3. Burn subtitles into video using FFmpeg subtitles filter:
  //    ffmpeg -i clip.mp4 -vf "subtitles=subs.srt:force_style=
  //    'FontName=Arial,FontSize=18,PrimaryColour=&Hffffff,
  //    OutlineColour=&H000000,Outline=2,Bold=1,Alignment=2'"
  //    -c:a copy output_subbed_<jobId>.mp4

  // Subtitle style: white bold text, black outline, bottom-center
  // Font size adapts to video resolution (720p → 18, 1080p → 22)

  const finalPath = `/tmp/final_${jobId}.mp4`;
  return finalPath;
}
```

## JOB PROGRESS TRACKING

Use Bull queue for the entire pipeline. Emit progress at each step:
```javascript
job.progress(10)  // Video downloaded
job.progress(30)  // Audio extracted
job.progress(50)  // Transcription done
job.progress(65)  // AI clip detection done
job.progress(80)  // Video cut done
job.progress(95)  // Subtitles burned
job.progress(100) // Ready for download
```

Store job result in DB:
```sql
-- Add to existing schema:
clips (
  id UUID PRIMARY KEY,

  clip_end FLOAT,
  ai_reason TEXT,               -- why Claude chose this clip

  final_file_path TEXT,
  file_size_bytes BIGINT,
  status VARCHAR,               -- processing | ready | downloaded | deleted
  created_at TIMESTAMP,
  downloaded_at TIMESTAMP
)
```

### Key behaviors:
- Progress bar polls GET /api/clipper/status/:jobId every 2 seconds
- Preview player uses HTML5 <video> streaming from /api/clipper/preview/:clipId
- Download button hits /api/clipper/download/:clipId
  → Server sets header: Content-Disposition: attachment; filename="autoflow_clip.mp4"
  → File streams to user browser (do NOT redirect to S3 signed URL — stream through backend)
- "Send to Scheduler" button pre-fills the existing scheduler form with this clip
- After download, show: "Delete from server? [Yes, free space]" → calls DELETE endpoint

## CLEANUP & STORAGE RULES
- Temp files auto-delete after 24 hours (cron job)
- Max 3 clips stored simultaneously per free user
- Never store original source video permanently — only the final clip
- Add to existing cron jobs file:
